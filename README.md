# 🚗 Car Price Predictor (VS Code Edition)

A machine learning project that predicts the resale price of a used car,
built as plain Python files to run in **VS Code** (no notebook needed).

Trained on the real `quikr_car.csv` dataset (scraped Quikr used-car listings).

## Project structure
```
car_price_vscode/
├── data/
│   ├── quikr_car.csv           # raw dataset (as provided)
│   └── cleaned_car_data.csv    # generated by clean_data.py
├── models/
│   └── car_price_model.joblib  # generated by train_model.py
├── plots/                      # generated by explore_data.py
├── src/
│   ├── clean_data.py           # cleans the raw dataset
│   ├── explore_data.py         # saves exploration plots as PNGs
│   ├── train_model.py          # trains & saves the model
│   └── predict.py              # CLI tool to predict a price
├── main.py                     # runs the full pipeline end-to-end
├── requirements.txt
└── README.md
```

## Setup

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## How to run

### Option A — run everything at once
```bash
python main.py
```
This cleans the data, trains the model, and runs one example prediction.

### Option B — run each step yourself
```bash
python src/clean_data.py        # data/quikr_car.csv -> data/cleaned_car_data.csv
python src/explore_data.py      # saves charts to plots/
python src/train_model.py       # trains model -> models/car_price_model.joblib
python src/predict.py --name "Maruti Suzuki Alto" --company Maruti \
    --fuel Petrol --year 2017 --kms 35000
```

### Interactive prediction (no flags)
```bash
python src/predict.py
```
You'll be prompted to type in the car's details one by one.

## About the data
The raw `quikr_car.csv` is messy real-world scraped data:
- `Price` has `"Ask For Price"` entries and comma-formatted numbers
- `kms_driven` has `"kms"` text mixed into the numbers
- `year` has some junk/non-numeric values
- `fuel_type` has missing values

`clean_data.py` handles all of this and produces a tidy ~718-row dataset.

## Model
- **Algorithm:** Random Forest Regressor
- **Features:** car name, company/brand, fuel type, car age, kilometers driven
- **Categorical encoding:** One-Hot Encoding (via `sklearn.compose.ColumnTransformer`)
- **Typical performance:** R² ≈ 0.5–0.6, MAE ≈ ₹1.2–1.5 lakh

> Real scraped data is noisy, so accuracy is lower than a clean synthetic
> dataset — that's expected and a good talking point if this is for coursework
> (mention the cleaning steps and feature limitations as next steps for improvement).

## Ideas to improve accuracy
- Add more features if available (engine size, location, transmission, owner count)
- Try Gradient Boosting / XGBoost instead of Random Forest
- Group rare car names/companies into an "Other" bucket to reduce overfitting
- Remove more outliers / cap extreme prices
